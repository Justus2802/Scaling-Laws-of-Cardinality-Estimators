The graph generation uses the signature vector and generates a synthetic graph. This makes it the heart of our generator. The pipeline get's a graph signature object which can either be generated by measuring from a real graph or be a synthetical signature. The only part the graph generator uses are the components of the signature vector saved it this signature object. 

The generators design consists of three stages. 
Stage 1 can be interpreted as an preprocessing step that uses the condensed signature values and extrapolates from them a internal schema representation. The schema is a intermediary step betweeen signature and instantiated graph that describes the graph closely but does not generate any object yet. From this a second stage is responsible for instantiating a full graph that fulfills the relation and node level statistical constraints the input schema passed from stage 1. For this different steps of sampling and instantiation are chained. In the third stage rewiring steered with a loss function targets structural constraints such as motif counts or clustering. Importantly this algorithm does not break the statistics from stage 2.
