# Guides for developing in this repo
- write brief comments, where it's not directly evident what is happening 
- write docstrings in python format
- update the documentation in @user_docs (usage-facing) and @developer_docs (design rationale,
  investigations, plans, course artifacts) after all edits — user_docs is for people using the
  package, developer_docs for people extending or researching it
- log uncommitted work in @CHANGELOG.md as you go, per the workflow described there
- docs should always descirbe the current state, not what was in the past (except for docs that explixitley have this objective/ plannign docs)
